# 直营门店分销缺口看板

## 项目简介

根据直营门店销量数据和分销标准，自动计算每个负责人所管门店的分销缺口（门店应该卖但没在卖的产品），以移动端网页看板展示，部署在 GitHub Pages 供团队查看。

## 文件结构

```
店铺分配新/
├── build_data.py               # 数据处理脚本（核心）
├── index.html                  # 看板页面（上传 GitHub Pages）
├── data.json                   # 生成的缺口数据（上传 GitHub Pages）
├── 南山直营门店销量-0616.xlsx    # 销量数据源（不上传）
├── 深圳类直营概况汇报.xlsx       # 分销标准数据源（不上传）
└── README.md                   # 本文档
```

## 数据来源

### 销量表：`南山直营门店销量-0616.xlsx`

每行是一条门店-产品记录。脚本用到的列：

| 列名 | 用途 |
|------|------|
| 标准客户名称 | 门店名 |
| 标准客户类型(整合) | 门店类型（商超A/B、生鲜店...） |
| 物料名称 | 产品名 |
| 负责人员 | 负责人 |

### 标准表：`深圳类直营概况汇报.xlsx` → `分销标准` sheet

一个产品×门店类型的矩阵，**标"1"表示该产品需要在该类型门店分销**。

列 C-I 分别是：商超AB、生鲜店、小超市、农贸店、粮油店、食杂店、便利店。

部分单元格有合并标记 `2选1`/`3选1`，表示该组产品中只需选 1 个分销。

## 核心逻辑

### 1. 产品名匹配

两个表的产品命名格式不同，通过 `extract_core_name()` 提取核心名后比对：

- 去掉包装数量（`*15`、`×4`）
- 去掉包装类型（`(纸箱)`、`(玻璃瓶)`）
- 去掉工艺等级描述（`/压榨/一级/`）
- 去掉规格数字（`900ML`、`5L`、`25KG`）

**只看品牌+品类名，不看规格。**
例如 `金龙鱼芝麻香油400ML*12(玻璃瓶)` 和 `金龙鱼芝麻香油220ML` 核心名都是 `金龙鱼芝麻香油`，视为同一产品。

### 2. 门店类型映射

| 销量表类型 | 映射到标准表列 |
|-----------|-------------|
| 商超A、商超B | 商超AB |
| 农贸菜场店 | 农贸店 |
| 生鲜店 | 生鲜店 |
| 小超市/粮油店/食杂店/便利店 | 不变 |
| 流通批发/批发流通渠道/其它客户 | 无对应标准，标注"无要求" |

### 3. 缺口判定

对每家门店，遍历该门店类型对应的标准产品列表：

- 标 `1` 的产品 → 必须分销，门店无此产品则计入缺口
- 标 `2选1`/`3选1` 的组 → 组内覆盖 1 个即算满足，全无则计为一条缺口
- 无分销标准的门店（流通批发等）→ 不计缺口

## 使用流程

### 更新数据

1. 将新的销量 Excel 和标准 Excel 放到脚本同级目录
2. 修改 `build_data.py` 底部 `build_data()` 中的文件名（如日期变化）：
   ```python
   sales_file = os.path.join(BASE_DIR, '南山直营门店销量-0616.xlsx')
   std_file = os.path.join(BASE_DIR, '深圳类直营概况汇报.xlsx')
   ```
3. 双击运行 `build_data.py`（或在终端 `python build_data.py`）
4. 检查终端输出的统计数字是否合理

### 本地预览

```bash
cd "店铺分配新"
python -m http.server 8080
```
浏览器打开 `http://localhost:8080`

### 部署到 GitHub Pages

1. 在 GitHub 创建仓库
2. 上传 `index.html` 和 `data.json` 到仓库根目录
3. Settings → Pages → Source: main branch, root → Save
4. 等待部署完成，访问 `https://用户名.github.io/仓库名/`

## 维护说明

### 修改脚本的行为

- **门店类型映射调整**：修改 `build_data.py` 中的 `STORE_TYPE_MAP` 字典
- **产品名匹配优化**：修改 `extract_core_name()` 函数，添加新的 SKIP_TOKENS 或 TRAILING_WORDS
- **新增"N选1"分组**：在分销标准 Excel 中将单元格合并并填写 `2选1` 或 `3选1`，脚本自动识别

### 修改看板样式

`index.html` 是纯 HTML/CSS/JS，可直接修改：
- CSS 在 `<style>` 标签内
- 页面结构在 JS 的 `renderOverview()` 和 `renderManagerDetail()` 函数中
- 规则说明在 `renderRules()` 函数中

### 注意事项

- 销量表和标准表的列名如果改了，脚本会自动报错提示找不到列，届时需更新 `find_col()` 里的列名
- 数据文件编码需为 UTF-8，Windows 下用 Excel 另存时注意
- 脚本按表头名匹配列，列顺序变化不影响运行
